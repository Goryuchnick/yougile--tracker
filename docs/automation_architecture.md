# Архитектура автоматизации YouGile (Маркетинг и Трансформация)

Этот документ описывает техническую реализацию трех выбранных сценариев автоматизации для экосистемы YouGile.

---

## 1. Динамическая приоритизация с ИИ (Сценарий #6)
**Цель:** Автоматическая сортировка задач в бэклоге на основе их важности, дедлайнов и содержания.

### Технический дизайн
*   **Скрипт:** `ai_prioritizer.py`
*   **Запуск:** Ежедневно по расписанию (Cron / Task Scheduler) или по вебхуку (при создании новой задачи).
*   **Алгоритм:**
    1.  **Сбор данных:** Получить все задачи из колонки "Бэклог" (или "Входящие").
    2.  **Анализ (LLM):** Отправить заголовок, описание и текущие стикеры в LLM (OpenAI/Claude).
        *   *Промпт:* "Оцени важность этой задачи от 1 до 10 для маркетингового отдела, учитывая дедлайн..."
    3.  **Действие:**
        *   Обновить стикер "Приоритет" (Высокий/Средний/Низкий).
        *   Переместить задачу в начало или конец колонки.
        *   Если задача критическая, добавить стикер "🔥".

### Необходимые ресурсы
*   API ключ YouGile (уже есть).
*   API ключ LLM провайдера (OpenAI / Anthropic).
*   ID стикеров для приоритетов.

## 2. Синхронизация Базы Знаний (Сценарий #7)
**Цель:** Автоматическое сохранение ценных инсайтов из закрытых задач в базу знаний.

### Технический дизайн
*   **Скрипт:** `knowledge_base_sync.py`
*   **Запуск:** Вебхук на событие `task.completed` или периодический поллинг "недавно завершенных" задач.
*   **Алгоритм:**
    1.  **Фильтр:** Проверять задачи только из проектов "Разработка" или "Маркетинг".
    2.  **Извлечение:** Сканировать описание и последние комментарии.
    3.  **Анализ (LLM):** "Есть ли в этом тексте полезное решение проблемы или инструкция? Если да, оформи как статью Markdown."
    4.  **Публикация:**
        *   *Вариант А (Простой):* Создать задачу на доске "База Знаний" с тегом "Статья".
        *   *Вариант Б (Продвинутый):* Отправить в Notion/Obsidian через их API.

### Интеграция
*   Рекомендуется начать с *Варианта А* (все внутри YouGile) для простоты.

## 3. Telegram-бот "Голос в Задачу" (Сценарий #10)
**Цель:** Создание задач на лету через голосовые сообщения.

### Технический дизайн
*   **Скрипт:** `voice_to_yougile_bot.py` (долгоживущий процесс).
*   **Интерфейс:** Telegram Bot.
*   **Алгоритм:**
    1.  **Прием:** Пользователь отправляет голосовое сообщение боту.
    2.  **Транскрибация:** Бот скачивает файл и отправляет в OpenAI Whisper API.
    3.  **Осмысление:** Текст отправляется в LLM для выделения Заголовка и Сути.
    4.  **Создание:** Бот создает задачу в колонке "Входящие" через YouGile API.
    5.  **Ответ:** Бот присылает ссылку на созданную задачу.

### Безопасность
*   Whitelist пользователей (чтобы никто чужой не создавал задачи).

1.  **Настройка окружения:** `requirements.txt` с библиотеками `openai`, `python-telegram-bot` и др.

4.  **Синхронизация Знаний (Сценарий #7):** Настройка правил фильтрации.
