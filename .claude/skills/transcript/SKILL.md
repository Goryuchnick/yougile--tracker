# Обработка транскрипта созвона

Извлеки задачи из транскрипта совещания и создай их в YouGile.

## Вход
- `$ARGUMENTS` — путь к .txt файлу с транскриптом или текст вставленный напрямую
- Если аргумент не указан — попроси пользователя вставить текст транскрипта

## Промпт для Gemini (бесплатный)

Используй Gemini API для извлечения задач. Промпт:

```
Ты — опытный проект-менеджер. Проанализируй транскрипт совещания
и извлеки ВСЕ задачи, поручения и обязательства.

Для каждой задачи определи:
- "title" — краткое название (до 80 символов)
- "description" — описание с контекстом из совещания
- "assignee" — кто ответственный (имя или "не назначен")
- "deadline" — дата YYYY-MM-DD если упомянута, иначе null
- "priority" — "High"/"Medium"/"Low"
- "checklist" — список подшагов

Правила:
- Извлекай ТОЛЬКО действия: "я сделаю...", "нужно...", "давайте..."
- Сегодняшняя дата: {current_date}
- Если дедлайн относительный — вычисли дату

Верни JSON массив.
```

## Маппинг на YouGile
- priority → стикер "Приоритет" (High=8ced62e1d595, Medium=55e6b0a1cb68, Low=414cda413f0a)
- deadline → поле `deadline` (timestamp ms)
- checklist → поле `checklists`
- assignee → поле `assigned` (нужен маппинг имя→user ID через GET /users)

## Процесс
1. Прочитай/получи транскрипт
2. Отправь в Gemini API с промптом выше
3. Распарси JSON-ответ
4. Покажи пользователю список задач для подтверждения
5. После подтверждения — создай задачи в YouGile через API
6. Выведи отчёт: какие задачи созданы, с какими ID

## Модель
Gemini 2.0 Flash (бесплатный tier): `gemini-2.0-flash`
Замена после 01.06.2026: `gemini-2.5-flash-lite`
